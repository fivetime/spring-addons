# 从 6.x 迁移到 7.0.0 的迁移指南

## 依赖项
- 将所有 spring-addons starter 替换为 `com.c4-soft.springaddons:spring-addons-starter-oidc`
- 将所有 spring-addons test starter 替换为 `com.c4-soft.springaddons:spring-addons-starter-oidc-test`
- 根据实际需求，酌情添加以下依赖：
  * `org.springframework.boot:spring-boot-starter-oauth2-resource-server`：用于通过 access token 保护的 REST API
  * `org.springframework.boot:spring-boot-starter-oauth2-client`：将 `spring-cloud-gateway` 配置为 BFF，或使用 Thymeleaf 等框架暴露服务端渲染模板时使用
  * 以上两者均添加：当同时对外暴露通过 access token 保护的 REST API 和通过 session 保护的其他资源时使用

## Java 源码

### 主代码
- 将 `SpringAddonsSecurityProperties` 重命名为 `SpringAddonsOidcProperties`。如果使用了嵌套属性，同时重命名：
  * `getIssuers()` → `getOps()`
  * `getLocation()` → `getIss()`
- 将 `SpringAddonsOAuth2ClientProperties` 替换为 `SpringAddonsOidcProperties::getClient`（只有 `SpringAddonsOidcProperties` 可以被自动注入）
- 整理 import

### 测试代码
- 将 `@AutoConfigureAddonsSecurity` 替换为 `@AutoConfigureAddonsMinimalSecurity`
- 将 `@AutoConfigureAddonsWebSecurity` 替换为以下之一：
  * `@AutoConfigureAddonsWebmvcSecurity`
  * `@AutoConfigureAddonsWefluxSecurity`

## 应用配置属性
这可能是迁移中最繁琐的部分。好在 IDE 的自动补全和语法高亮应该能提供帮助。

### 通用配置
- 将 `com.c4-soft.springaddons.security` 重命名为 `com.c4-soft.springaddons.oidc`
- 将 `issuers` 重命名为 `ops`（代表 OpenID Providers，`com.c4-soft.springaddons.security.issuers` 变为 `com.c4-soft.springaddons.oidc.ops`）
- 将 OpenID Provider 的 `location` 重命名为 `iss`：若已设置，该值用于向 JWT decoder 添加"issuer"校验器（校验 token 的 `iss` claim），`com.c4-soft.springaddons.security.issuers[].location` 变为 `com.c4-soft.springaddons.oidc.ops[].iss`
- 将 `audience` 重命名为 `aud`：若已设置，该值用于向 JWT decoder 添加"audience"校验器（校验 token 的 `aud` claim），`com.c4-soft.springaddons.security.issuers[].aud` 变为 `com.c4-soft.springaddons.oidc.ops[].aud`

CORS 配置在 client 和 resource server 上均有改进：改用 `allowed-origin-patterns` 替代 `allowed-origins`。这是使用 `allow-credentials` 的必要条件，同时也更灵活——可以定义 ant 风格的模式，如 `https://*.my-domain.pf`。
- 将 `allowed-origins` 重命名为 `allowed-origin-patterns`
- 如有需要，可添加 `allow-credentials` 和 `max-age`（这是新增的配置选项）

### Resource Server
Resource server 的 `Security(Web)FilterChain` 现在可以通过 `com.c4-soft.springaddons.security.resourceserver.enabled=false` 完全禁用。

Resource server 专属属性已归入新的 `resourceserver` 子节点：
- 将 `cors` 下移一级到 `resourceserver` 下（`com.c4-soft.springaddons.security.cors` 变为 `com.c4-soft.springaddons.oidc.resourceserver.cors`）
- 将 `permit-all` 下移一级到 `resourceserver` 下（`com.c4-soft.springaddons.security.permit-all` 变为 `com.c4-soft.springaddons.oidc.resourceserver.permit-all`）

### Client
- 将 `allowed-origins` 重命名为 `allowed-origin-patterns`（`com.c4-soft.springaddons.security.client.cors.allowed-origins` 变为 `com.c4-soft.springaddons.security.client.cors.allowed-origin-patterns`）
- `oauth2-logout` 现在是以 client registration ID 为键的 map，而非数组。将原有条目的 `client-registration-id` 作为键，其余属性保留。例如：
```yaml
          oauth2-logout:
            - client-registration-id: cognito-confidential-user
              uri: https://spring-addons.auth.us-west-2.amazoncognito.com/logout
              client-id-request-param: client_id
              post-logout-uri-request-param: logout_uri
            - client-registration-id: auth0-confidential-user
              uri: ${auth0-issuer}v2/logout
              client-id-request-param: client_id
              post-logout-uri-request-param: returnTo
```
变为：
```yaml
          oauth2-logout:
            cognito-confidential-user:
              uri: https://spring-addons.auth.us-west-2.amazoncognito.com/logout
              client-id-request-param: client_id
              post-logout-uri-request-param: logout_uri
            auth0-confidential-user:
              uri: ${auth0-issuer}v2/logout
              client-id-request-param: client_id
              post-logout-uri-request-param: returnTo
```
其中 `cognito` 和 `auth0` 分别是 `spring.security.oauth2.client.registration.cognito-confidential-user.provider` 和 `spring.security.oauth2.client.registration.auth0-confidential-user.provider` 的值。
